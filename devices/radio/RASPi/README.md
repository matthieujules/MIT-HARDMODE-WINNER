# Radio Raspberry Pi Runtime

This folder contains the Pi-native runtime for the Radio device.

It is designed to run without any browser UI:

- receives a text command from the control plane or CLI
- reuses the shared radio planner in `devices/radio/brain.py`
- turns the continuous-rotation dial through a GPIO pin
- plays audio locally through the Raspberry Pi's USB speaker

## Files

- `main.py` — entry point for local testing and device boot
- `runtime.py` — executes the shared radio plan on Pi hardware
- `audio.py` — local audio playback for music and generated podcast clips
- `dial.py` — continuous-rotation servo control over GPIO
- `config.py` — loads Pi runtime settings from `devices/radio/config.yaml`

## Install

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r devices/radio/RASPi/requirements.txt
```

## Run

```bash
python3 devices/radio/RASPi/main.py "play calm morning music"
python3 devices/radio/RASPi/main.py --loop
```

## Notes

- Music playback prefers local files in `devices/radio/media/`.
- If a local file match is not found and a remote preview URL exists, the runtime can download the preview and play it locally.
- Podcast/TTS playback uses audio files generated by `devices/radio/brain.py`.
- The continuous dial is speed-based, so `devices/radio/config.yaml` includes PWM values and turn duration rather than target angles.
- Supported local audio players are tried in this order: `mpg123`, `ffplay`, `cvlc`, then `aplay` for WAV files.
